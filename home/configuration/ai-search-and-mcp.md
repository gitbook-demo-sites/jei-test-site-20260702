# AI search and MCP

AI answers are only as good as the structure and language of the source pages.

## Make content retrieval-friendly

| Practice | Why it helps |
| --- | --- |
| Use direct headings | AI can identify the answer boundary. |
| Define terms once | Reduces conflicting answers. |
| Keep procedures step-based | Makes task answers easier to cite. |
| Put caveats near the action | Avoids missing important constraints. |
| Avoid placeholder pages | Prevents weak answers from low-value content. |

## Example answer shape

When writing a page, imagine the AI answer a reader should get:

> To publish the test site, open the site preview, check navigation, verify visibility is set to share-link, then publish.

If the page cannot produce that kind of answer, it probably needs clearer headings or a stronger checklist.
