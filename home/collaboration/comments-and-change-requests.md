# Comments and change requests

Comments are best for local feedback. Change requests are best for reviewing a coherent set of edits.

## Use comments when

* A sentence is unclear.
* A link may be wrong.
* A reviewer has a small question.
* A page needs a factual check.

## Use change requests when

* Several pages change together.
* A page is being rewritten.
* Publishing should wait for approval.
* The change affects navigation or permissions.

{% hint style="success" %}
The cleanest review habit is to keep each change request focused on one outcome.
{% endhint %}
