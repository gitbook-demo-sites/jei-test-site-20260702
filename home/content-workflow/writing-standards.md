# Writing standards

Good documentation helps a reader finish a task with confidence. Keep each page focused on one reader goal.

## Page structure

Use this shape for most pages:

1. State the job the page helps with.
2. Give the shortest useful explanation.
3. Put actions in clear steps.
4. Add examples where mistakes are likely.
5. End with related links or a checklist.

## Style rules

| Rule | Why it matters |
| --- | --- |
| Use task-based titles | Readers scan for outcomes, not internal labels. |
| Keep paragraphs short | Search, AI answers, and human readers all benefit. |
| Put warnings near the risky step | Warnings are useful only where decisions happen. |
| Use consistent names | Mixed terminology weakens search and AI retrieval. |

{% hint style="warning" %}
Avoid pages that only say "coming soon." They create weak search results and reduce trust. Hide unfinished pages until they have a useful first version.
{% endhint %}
