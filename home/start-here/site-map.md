# Site map

The site is organized around the workflows a GitBook user is likely to test.

| Section | Purpose |
| --- | --- |
| Start here | First-time orientation and quick testing. |
| Content workflow | Writing, review, and content quality habits. |
| Publishing | Visibility, share links, and launch readiness. |
| Collaboration | Roles, comments, and change requests. |
| Configuration | Branding, navigation, AI search, and MCP. |
| Reference | Checklists and answers that should stay stable. |

## When to add a new page

Add a page when the topic has a distinct job to be done. If it is a small detail, add it to an existing page or use an expandable section instead.

<details>
<summary>Example</summary>

Do not create separate pages for every button in the editor. Create one `Writing standards` page, then add examples for buttons, hints, tabs, and tables inside it.
</details>
