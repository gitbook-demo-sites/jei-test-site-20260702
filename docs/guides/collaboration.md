# Collaboration

Orbitly is built for teams. Here's how to work together effectively.

## Comments & mentions

Every mission has a comment thread. Use `@name` to mention a teammate — they'll get a notification via their preferred channel (in-app, email, or Slack).

Comments support full markdown, including code blocks and file attachments up to 25 MB.

## Reviews

Request a review on any mission before marking it done:

1. Open the mission and click **Request Review**
2. Pick one or more reviewers
3. Reviewers can **Approve** or **Request Changes**

Missions with pending reviews show an orange badge on the board.

## Shared views

Save any filtered board view and share it:

* **My open missions** — `assignee:me status:open`
* **This week's launches** — `window:current status:done`
* **Blocked work** — `label:blocked`

Shared views appear in the sidebar for everyone in the project.

## Notifications

By default you're notified when:

* You're mentioned or assigned
* A mission you own changes status
* A review is requested from you

Tune this per-project under **Settings → Notifications**. We recommend enabling the **Daily Digest** and disabling per-event emails for busy projects.

## Guest access

Invite external collaborators (clients, contractors) as **Guests**. Guests only see projects they're explicitly added to and never see workspace-level settings or member lists.
